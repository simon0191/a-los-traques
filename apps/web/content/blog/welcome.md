Este es el primer post de este sitio. Aquí iremos documentando la evolución
técnica del juego: decisiones de arquitectura, pequeños bugs memorables, el
porqué de ciertas mecánicas.

Ahora mismo, el blog es estático: los posts son archivos Markdown dentro del
repo. Sin CMS, sin base de datos detrás, sin dashboard de edición. Si alguna vez
se necesita algo más, lo cambiaremos entonces.
